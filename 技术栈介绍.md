# 技术栈介绍

## 概述

本项目采用 **Monorepo** 架构，使用现代化前端技术栈，提供高效的开发体验和优秀的可维护性。

---

## 核心架构

### Monorepo + pnpm + Turborepo

#### Monorepo (单体仓库)
- **定义**: 多个项目（packages）放在同一个代码仓库中管理
- **优势**:
  - 代码共享便捷
  - 统一版本管理
  - 跨项目重构更简单
  - 集中式 CI/CD

#### pnpm ( performant npm )
- **定义**: 高性能、节省空间的包管理器
- **优势**:
  - **节省磁盘空间**: 采用内容寻址存储，相同版本的包只存储一份
  - **安装速度快**: 并行安装，效率更高
  - **严格的依赖管理**: 避免幽灵依赖问题
  - **兼容 npm**: 全面支持 npm 的 package.json 格式

#### Turborepo
- **定义**: 高性能的增量构建系统，专为 Monorepo 设计
- **优势**:
  - **增量构建**: 只重新构建有变更的部分
  - **任务调度**: 智能管理任务依赖和执行顺序
  - **远程缓存**: 支持团队共享构建缓存（需要登录）
  - **可视化**: 直观展示任务执行情况

---

## 后端 / 数据层

### Supabase
- **定义**: 开源的 Firebase 替代品，基于 PostgreSQL
- **核心功能**:
  - **PostgreSQL 数据库**: 强大的关系型数据库
  - **实时订阅**: 数据库变更实时推送到客户端
  - **身份认证**: 内置用户注册、登录、OAuth 等
  - **文件存储**: 图片、文件上传和托管
  - **Edge Functions**: 在边缘节点运行的 Serverless 函数
  - **自动生成 API**: 基于数据库自动生成 RESTful 和 GraphQL API
- **优势**:
  - 开源可控，无供应商锁定
  - 与 Next.js 集成良好
  - 提供免费额度开箱即用

---

## 前端框架

### Next.js
- **定义**: React 生态中最流行的全栈框架，由 Vercel 开发
- **核心特性**:
  - **App Router**: 基于文件系统的路由，支持 layouts、nested routes
  - **Server Components**: 服务端组件，减少客户端 JavaScript 体积
  - **Server Actions**: 在服务端直接操作数据库
  - **静态站点生成 (SSG)**: 预渲染静态页面
  - **增量静态再生成 (ISR)**: 无需全站重建即可更新静态内容
  - **自动代码分割**: 每个页面只加载必要的代码
  - **内置优化**: 图片优化、字体优化、脚本加载策略

---

## UI 组件库

### shadcn/ui
- **定义**: 不是传统意义上的组件库，而是一组**可复制粘贴的组件源码**
- **核心理念**:
  - 组件代码直接放入项目中，可完全自定义
  - 不作为 npm 依赖包分发
  - 基于 Radix UI 原语构建
- **优势**:
  - **完全可控**: 源码在手，可任意修改
  - **无依赖锁定**: 不受第三方包版本更新影响
  - **Tree-shaking 友好**: 只使用需要的组件
  - **设计一致**: 与 Tailwind CSS 完美配合

### Tailwind CSS
- **定义**: 原子化 CSS 框架，通过工具类组合构建样式
- **核心特性**:
  - **即时设计**: 直接在 HTML 中编写样式
  - **响应式设计**: 简洁的响应式断点前缀
  - **暗色模式**: 优雅的暗色模式支持
  - **自定义设计系统**: 通过配置文件定义设计令牌
- **优势**:
  - 开发速度快
  - 包体积小（只打包使用的样式）
  - 一致性好

---

## React

### React
- **定义**: 用于构建用户界面的 JavaScript 库
- **核心概念**:
  - **组件化**: 一切皆组件
  - **虚拟 DOM**: 高效的 DOM diff 算法
  - **Hooks**: 函数组件的状态和副作用管理
  - **单向数据流**: 数据从父组件流向子组件
- **生态优势**:
  - 最大的社区生态
  - 丰富的第三方库
  - 完善的开发工具链

│   └── web/                 # Next.js 主应用
│       ├── src/

        ├── src/

## 总结

| 分类 | 技术 | 作用 |
|------|------|------|
| 架构管理 | Monorepo + pnpm + Turborepo | 统一管理多项目，提升构建性能 |
| 后端服务 | Supabase | 数据库、认证、存储、Serverless |
| 前端框架 | Next.js | SSR/SSG、服务端能力 |
| UI 框架 | React | 组件化 UI 开发 |
| 组件库 | shadcn/ui | 可定制的高质量组件 |
| 样式方案 | Tailwind CSS | 原子化 CSS，快速构建样式 |

这套技术栈组合提供了:
- **开发效率**: Monorepo + Turborepo 优化构建流程
- **开发体验**: Next.js App Router + React 18 新特性
- **可维护性**: TypeScript 类型安全 + 代码共享
- **可扩展性**: Supabase 承担后端复杂度
