# 签到系统 - 项目设计文档

## 一、项目概述

### 1.1 项目简介
一个简洁的 **注册登录签到系统**，用户完成注册登录后进入签到页面，每日可签到多次，签到记录持久化存储。

### 1.2 功能范围
| 模块 | 功能点 |
|------|--------|
| 用户模块 | 注册、登录、登录状态保持 |
| 签到模块 | 签到（不限次数，每次生成新记录）、签到状态展示、签到记录列表 |

### 1.3 设计风格
- **主色调**: 米白色系（#FFF8F0 / #FAF5EF）
- **风格特点**: 简约、温暖、轻盈
- **动效**: 按钮反馈、加载状态、成功/失败提示均有平滑过渡动画

## 三、数据库设计

### 3.1 Supabase 简介
- 开源 Firebase 替代品
- 基于 PostgreSQL
- 提供 Auth、Database、Storage 等服务

### 3.2 数据表设计

#### users 表（使用 Supabase Auth，内置）
Supabase Auth 自动管理用户表 `auth.users`，包含：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 用户唯一标识 |
| email | text | 邮箱 |
| created_at | timestamptz | 创建时间 |

#### check_ins 表（签到记录）
```sql
CREATE TABLE check_ins (
  id          UUID        DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id     UUID        NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  check_in_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 创建索引加速查询
CREATE INDEX idx_check_ins_user_id ON check_ins(user_id);
CREATE INDEX idx_check_ins_check_in_at ON check_ins(check_in_at);

-- RLS 策略：用户只能查看自己的签到记录
ALTER TABLE check_ins ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own check_ins"
  ON check_ins FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own check_ins"
  ON check_ins FOR INSERT
  WITH CHECK (auth.uid() = user_id);
```

#### 签到记录表结构
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 记录唯一标识 |
| user_id | UUID | 用户ID（关联 auth.users） |
| check_in_at | TIMESTAMPTZ | 签到时间 |
| created_at | TIMESTAMPTZ | 记录创建时间 |

### 4.3 动效设计

- Hover: scale(1.02), 背景色加深, 200ms ease-out
- Active: scale(0.98), 150ms
- Loading: 旋转图标 + 禁用状态

#### 表单反馈
- 成功: 绿色边框闪烁 + 勾选图标 + 200ms 后跳转
- 失败: 红色边框 + 抖动动画 (shake), 300ms

#### 签到按钮
- **未签到状态**: 黄色按钮 (#F5C842 / #E8B830)，提示"今日未签到"
- **已签到状态**: 蓝色按钮 (#4A90D9 / #3B7DD8)，提示"已签到"
- 点击: 波纹扩散效果 (ripple)
- 签到成功: 短暂放大 + 弹出成功提示
- **注意**: 无论是否已签到，点击都会插入新记录，只是按钮颜色根据当日状态变化

#### 页面过渡
- 路由切换: fade + slide, 300ms

## 五、接口设计

### 5.1 Supabase API

本项目直接使用 Supabase 客户端库调用后端，无需手动编写 REST API。

| 操作 | 方法 | 说明 |
|------|------|------|
| 注册 | `supabase.auth.signUp()` | 创建用户 |
| 登录 | `supabase.auth.signInWithPassword()` | 登录验证 |
| 退出 | `supabase.auth.signOut()` | 清除会话 |
| 获取会话 | `supabase.auth.getSession()` | 获取当前用户 |

### 5.2 数据库操作 (签到)

| 操作 | 方法 | 说明 |
|------|------|------|
| 插入签到 | `supabase.from('check_ins').insert()` | 每次签到生成新记录 |
| 查询签到 | `supabase.from('check_ins').select()` | 获取签到列表（倒序） |
| 查询今日签到 | 带 `check_in_at` 日期过滤 | 判断今日是否已签到（用于按钮颜色） |

### 5.3 典型调用示例

```typescript
// 登录
const { data, error } = await supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'password123'
});

// 签到（每次点击都插入新记录）
const { data, error } = await supabase
  .from('check_ins')
  .insert({ user_id: user.id })
  .select();

// 获取签到记录（倒序，最新在前）
const { data, error } = await supabase
  .from('check_ins')
  .select('*')
  .order('check_in_at', { ascending: false });

// 查询今日是否已签到（用于判断按钮颜色）
const today = new Date().toISOString().split('T')[0];
const { data: todayCheckins } = await supabase
  .from('check_ins')
  .select('id')
  .eq('user_id', user.id)
  .gte('check_in_at', `${today}T00:00:00`)
  .lt('check_in_at', `${today}T23:59:59`)
  .limit(1);

const hasCheckedInToday = todayCheckins && todayCheckins.length > 0;
```

### 6.1 路由流程

```
       │ 是
       ▼
┌─────────────┐
│   签到页面   │
└─────────────┘
```

### 6.2 用户交互流程

#### 注册流程
1. 用户输入邮箱、密码
2. 点击注册按钮
3. 显示加载状态
4. 成功 → 显示成功动画 → 跳转登录页
5. 失败 → 显示错误动画 + 错误信息

#### 登录流程
1. 用户输入邮箱、密码
2. 点击登录按钮
3. 显示加载状态
4. 成功 → 显示成功动画 → 跳转签到页
5. 失败 → 显示错误动画 + 错误信息

#### 签到流程
1. 进入签到页，查询今日是否有签到记录
2. 显示实时时间
3. 根据今日签到状态显示按钮颜色（黄/蓝）
4. 点击签到按钮
5. 显示加载状态
6. 成功后：
   - 插入新签到记录到数据库
   - 按钮保持或变为蓝色
   - 新记录出现在下方列表顶部
7. 刷新签到记录列表

#### 签到规则
- **不限制签到次数**: 每次点击签到都会在数据库中生成一条新记录
- **当日状态判断**: 根据 `check_in_at` 日期判断今日是否已签到（只查最早一条）
- **按钮颜色**:
  - 黄色: 今日尚未签到
  - 蓝色: 今日已有签到记录
